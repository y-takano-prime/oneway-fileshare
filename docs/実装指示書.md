## 技術仕様

- **フレームワーク:** Laravel 8（PHP 7.4）
- **フロントエンド:** Bootstrap 5（CDN）、バニラJS（jQueryは使用しない）
- **メール:** MAIL_MAILER=log（開発環境）
- **ファイル保存:** `storage/app/uploads/{user_id}/{uuid}.{拡張子}`

## UIデザイン方針

### 管理画面（ダッシュボード・ファイル管理・URL管理）
**Tabler**（https://tabler.io）を参考にすること。
- Bootstrap 5 ベースのクリーンな管理画面デザイン
- CDN で利用可能：`https://cdn.jsdelivr.net/npm/@tabler/core@latest/dist/css/tabler.min.css`
- サイドバー・カード・テーブルのレイアウトを活用する

### 相手先のダウンロード画面（passcode・otp・complete・error）
**WeTransfer**（https://wetransfer.com）を参考にすること。
- 画面中央に1つの操作だけを表示するシンプルな構成
- 余白を広く取り、迷わない1ステップ1アクションのUI
- `layouts/app.blade.php` は使用せず、専用のシンプルレイアウトで作成する

#### DownloadController.php（新規）
**パス:** `app/Http/Controllers/DownloadController.php`

各メソッド冒頭で以下を共通チェックする：
```
1. tokenでDownloadUrlを取得。なければ abort(404)
2. deleted_at がある → abort(404)
3. expires_at が過去 → view('download.error', ['reason' => 'expired'])
4. download_limit が設定済みで download_count >= download_limit → view('download.error', ['reason' => 'limit'])
```

```
showPasscode($token): 共通チェック→view('download.passcode')
verifyPasscode($token, Request):
  passcodeが設定されている場合はhash_equals検証
  OKならセッションに passcode_verified=true を保存→verifyEmailへ
  NGならエラー付きでview('download.passcode')

verifyEmail($token, Request):
  セッションのpasscode_verified確認
  recipient_emailと一致確認
  OKならAuthCodeを生成（6桁・10分有効）してOtpMailを送信
  セッションに email_verified=true を保存→view('download.otp')
  NGならエラー付きでview('download.passcode')

showOtp($token):
  セッションのemail_verified確認
  lock_until確認（ロック中ならエラー表示）
  view('download.otp')

verifyOtp($token, Request):
  セッションのemail_verified確認
  AuthCodeの最新レコードを取得
  lock_until確認
  expires_at確認（期限切れならエラー）
  code一致確認
    一致: used_at=now()・セッションにotp_verified=true→view('download.complete')
    不一致: failed_count+1・5回でlock_until=now()+30分

download($token):
  セッションのotp_verified確認
  Storage::download でファイルをレスポンス
  download_count+1
  AccessLogにaction='download'を記録
  notify_on_downloadがtrueならDownloadNotificationMailを送信
```

### Step 5: ルーティング

`routes/web.php` を以下の構成で作成する（既存内容は削除して上書き）：

```php
// ゲスト
Route::middleware('guest')->group(function () {
    Route::get('/login', [LoginController::class, 'showLoginForm'])->name('login');
    Route::post('/login', [LoginController::class, 'login']);
});

// 認証済み
Route::middleware('auth')->group(function () {
    Route::post('/logout', [LoginController::class, 'logout'])->name('logout');
    Route::get('/dashboard', [DashboardController::class, 'index'])->name('dashboard');
    Route::resource('files', FileController::class)->only(['index', 'store', 'destroy']);
    Route::resource('urls', DownloadUrlController::class)->only(['index', 'create', 'store', 'show', 'destroy']);

    // 管理者専用
    Route::middleware('admin')->prefix('admin')->name('admin.')->group(function () {
        Route::resource('users', UserController::class);
        Route::get('logs', [AccessLogController::class, 'index'])->name('logs.index');
        Route::get('settings', [SettingController::class, 'index'])->name('settings.index');
        Route::post('settings', [SettingController::class, 'update'])->name('settings.update');
    });
});

// 相手先（認証不要）
Route::prefix('d/{token}')->group(function () {
    Route::get('/', [DownloadController::class, 'showPasscode'])->name('download.passcode');
    Route::post('/passcode', [DownloadController::class, 'verifyPasscode'])->name('download.verify-passcode');
    Route::post('/email', [DownloadController::class, 'verifyEmail'])->name('download.verify-email');
    Route::get('/otp', [DownloadController::class, 'showOtp'])->name('download.otp');
    Route::post('/otp', [DownloadController::class, 'verifyOtp'])->name('download.verify-otp');
    Route::get('/download', [DownloadController::class, 'download'])->name('download.file');
});

// トップは /login へリダイレクト
Route::get('/', fn() => redirect()->route('login'));
```

### Step 7: Artisan Command

#### CleanupExpiredFiles.php（新規）
**パス:** `app/Console/Commands/CleanupExpiredFiles.php`
**コマンド名:** `fileshare:cleanup`

```
処理:
1. settings.jsonから cleanup_grace_days を取得（デフォルト7）
2. expires_at が (現在 - grace_days) より前の DownloadUrl を取得
3. notify_before_delete が ON かつ削除7日前なら DeleteNotificationMail を送信
4. 猶予期間超過分: Storage::delete でファイル削除→deleted_files_logに記録→DownloadUrl物理削除
```

`app/Console/Kernel.php` の schedule メソッドに追加：
```php
$schedule->command('fileshare:cleanup')->dailyAt('02:00');
```

## 注意事項

- PHP 7.4 非対応の構文は使用しないこと（named arguments・union types・match式のアロー関数以外の arrow fn など）
- `Str::random(64)` でトークンを生成すること
- パスコードは `Hash::make()` でハッシュ化して保存し、検証は `Hash::check()` を使うこと
- ファイルの保存先は `storage/app/uploads/` 配下とし `public/` 直下には置かないこと
- 削除ボタンはすべて確認ダイアログ付きにすること（`onclick="return confirm('削除しますか？')"` ）
