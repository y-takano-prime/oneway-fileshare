# ファイル授受システム 要件定義書

- **システム名:** oneway-fileshare
- **作成日:** 2026年6月15日
- **対象期間:** 約3ヶ月（プロトタイプ開発）

---

## 1. 目的

メール添付によるファイル授受の課題（迷惑メール判定・誤送信・容量制限・履歴管理の困難）を解消するため、ブラウザ上で安全にファイルを授受できるWebシステムを構築する。

**授受の方向:** 当社（担当者） → 相手先（片方向）
**相手先の操作:** ダウンロードのみ（アップロード機能なし）

---

## 2. 現状課題

| 課題 | 内容 |
|------|------|
| 迷惑メール判定 | ZIP・Word・Excel添付や長いURLが迷惑メール扱いされ、相手先に届かないケースが増加 |
| 誤送信リスク | 宛先・添付ファイル間違い、CC/BCCミスによる情報漏洩 |
| 容量制限 | メール添付は10〜25MB程度の制限があり、大容量ファイルに対応できない |
| 履歴管理が困難 | 誰が・いつ・どのファイルを取得したか把握しにくい |
| 個人情報管理 | 応募書類などの個人情報をメール経由で扱うリスク |

---

## 3. システム概要

### 利用フロー

1. 担当者が管理画面からファイルをアップロード
2. 担当者がダウンロード用URLを発行（有効期限・パスコード等を設定）
3. 担当者が相手先へURLをメールで通知
4. 相手先がブラウザからURLにアクセスしてファイルをダウンロード
5. 担当者が管理画面でダウンロード履歴を確認

### 利用シーン

**取引先向け**
- 契約書・仕様書・請求書などを安全に渡す

**求人応募者向け**
- 採用書類（内定通知・案内書類など）を渡す

## 4. 機能要件

### 4.1 ファイルアップロード機能（担当者向け）

- 担当者が管理画面からファイルをサーバーにアップロードできる
- 複数ファイルの同時アップロード対応
- ドラッグ＆ドロップ対応
- アップロード中の進捗表示

### 4.2 ダウンロードURL発行機能

- アップロードしたファイルに対してダウンロード用URLを発行できる
- URLごとに以下を設定可能
  - 有効期限（日時指定）
  - パスコード
  - 相手先メールアドレス（2段階認証用・必須）
  - ダウンロード可能回数の上限（任意）

### 4.3 ファイルダウンロード機能（相手先向け）

- 相手先がURLにアクセスしてファイルをダウンロードできる
- スマートフォンからも操作可能
- ダウンロード時にアクセスログを記録
- 有効期限切れ・ダウンロード上限超過時はエラー表示

#### 2段階認証フロー

1. 相手先がURLにアクセス
2. パスコードを入力
3. メールアドレスを入力（URL発行時に登録済みのアドレスと一致確認）
4. 一致した場合、登録メールアドレス宛にワンタイムコードを送信
5. ワンタイムコードを入力して認証完了
6. ダウンロード画面を表示

#### 認証仕様

- ワンタイムコードは6桁の数字
- コードの有効期限は10分
- 連続認証失敗（5回）でURLへのアクセスを一時ロック

### 4.4 通知メール機能

- 相手先がダウンロードした際に担当者へメール通知（任意設定）
- 通知内容：ダウンロード日時・IPアドレス・ファイル名

### 4.5 アクセスログ機能

- URLへのアクセス日時・IPアドレスを記録
- ファイルのダウンロード履歴を記録
- 管理画面から確認可能

### 4.6 ユーザー管理機能

- 管理者アカウントでログインできる
- 担当者ごとにアカウントを発行できる
- 担当者は自分が発行したURLの管理のみ可能（管理者は全件閲覧可能）

### 4.7 自動削除機能

- 有効期限が切れたURLおよびファイルを自動削除
- 削除実行前に一定の猶予期間を設ける（デフォルト：有効期限後7日）
- 削除前に担当者へメール通知（任意設定）
- 削除済みファイルの記録をログとして保持（ファイル名・削除日時）

### 4.8 セキュリティ強化機能

- URLトークンは推測困難な長さのランダム文字列を使用（64文字以上）
- パスコードを必須化する設定を管理者が選択可能
- ウイルスチェック機能（プロトタイプ後の本番化時に対応）

---

## 5. 非機能要件

| 項目 | 要件 |
|------|------|
| セキュリティ | HTTPS通信必須、パスコード認証、アクセスログ保存 |
| ファイルサイズ | 1ファイルあたり最大100MB（目安） |
| 対応ブラウザ | Chrome・Edge・Safari・Firefox 最新版 |
| スマホ対応 | レスポンシブデザイン対応（ダウンロード画面） |
| 可用性 | 社内環境（XAMPP）での運用を前提とする |

## 6. 技術構成

| 項目 | 採用技術 |
|------|------|
| フレームワーク | Laravel 8（PHP 7.4） |

| フロントエンド | Blade テンプレート + Bootstrap |

| メール送信 | SMTP（社内メールサーバー） |
| 開発環境 | XAMPP（Windows） |

## 7. 画面一覧

| 画面名 | 対象ユーザー | 説明 |
|--------|------------|------|
| ログイン画面 | 担当者・管理者 | メールアドレス＋パスワードでログイン |
| ダッシュボード | 担当者・管理者 | 発行URL一覧・ダウンロード状況の確認 |
| ファイルアップロード画面 | 担当者・管理者 | ファイルをサーバーにアップロード |
| URL発行画面 | 担当者・管理者 | 有効期限・パスコード等を設定してURLを発行 |
| パスコード入力画面 | 相手先（外部） | パスコードとメールアドレスを入力 |
| ワンタイムコード入力画面 | 相手先（外部） | メールで受信したコードを入力して認証 |
| ファイルダウンロード画面 | 相手先（外部） | 認証後にファイルをダウンロード |
| アクセスログ画面 | 管理者 | アクセス・ダウンロード履歴の確認 |
| ユーザー管理画面 | 管理者 | 担当者アカウントの追加・編集・削除 |

## 8. データベース設計（概要）

| テーブル名 | 主な項目 |
|-----------|---------|
| users | id, name, email, password, role, is_active, created_at |
| shared_files | id, user_id, original_name, stored_path, file_size, mime_type, created_at |
| download_urls | id, shared_file_id, user_id, token, passcode, recipient_email, expires_at, download_limit, download_count, notify_on_download, deleted_at |
| access_logs | id, download_url_id, ip_address, action, created_at |
| auth_codes | id, download_url_id, code, expires_at, used_at, failed_count, lock_until |
| deleted_files_log | id, original_name, stored_path, deleted_by, deleted_at |

## 11. 用語定義

| 用語 | 定義 |
|------|------|
| 担当者 | 社内でファイルをアップロード・URLを発行・管理するユーザー |
| 管理者 | システム全体を管理するユーザー（担当者権限も持つ） |
| 相手先 | URLを受け取ってファイルをダウンロードする外部ユーザー |
| トークン | URL発行時に生成されるランダムな文字列。URLの一部として使用 |
