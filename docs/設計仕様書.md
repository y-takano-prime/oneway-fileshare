## 3. モデル設計

### 3.1 User

**ファイル:** `app/Models/User.php`

```
fillable: name, email, password, role, is_active
hidden: password, remember_token
casts: is_active → boolean
role: 'admin' または 'staff'
```

**リレーション:**
- hasMany(SharedFile::class)
- hasMany(DownloadUrl::class)

### 3.2 SharedFile

**ファイル:** `app/Models/SharedFile.php`

```
fillable: user_id, original_name, stored_path, file_size, mime_type
```

**リレーション:**
- belongsTo(User::class)
- hasMany(DownloadUrl::class)

### 3.3 DownloadUrl

**ファイル:** `app/Models/DownloadUrl.php`

```
fillable: shared_file_id, user_id, token, passcode, recipient_email,
          expires_at, download_limit, download_count, notify_on_download
casts: expires_at → datetime, notify_on_download → boolean
uses: SoftDeletes
```

**リレーション:**
- belongsTo(SharedFile::class)
- belongsTo(User::class)
- hasMany(AccessLog::class)
- hasOne(AuthCode::class) ※最新のもの

**スコープ:**
- scopeValid($query): expires_at が未来、deleted_at が null、download_limit 未超過

### 3.4 AccessLog

**ファイル:** `app/Models/AccessLog.php`

```
fillable: download_url_id, ip_address, action
timestamps: false（created_atのみ）
action の値: access, passcode_ok, passcode_fail, email_ok, email_fail, otp_ok, otp_fail, download
```

**リレーション:**
- belongsTo(DownloadUrl::class)

### 3.5 AuthCode

**ファイル:** `app/Models/AuthCode.php`

```
fillable: download_url_id, code, expires_at, used_at, failed_count, lock_until
casts: expires_at → datetime, used_at → datetime, lock_until → datetime
timestamps: false（created_atのみ）
```

**リレーション:**
- belongsTo(DownloadUrl::class)

### 3.6 DeletedFilesLog

**ファイル:** `app/Models/DeletedFilesLog.php`

```
fillable: original_name, stored_path, deleted_by
timestamps: false（deleted_atのみ）
table: 'deleted_files_log'
```

## 4. コントローラー設計

### 4.1 Auth\LoginController

**ファイル:** `app/Http/Controllers/Auth/LoginController.php`

| メソッド | 処理内容 |
|---------|---------|
| showLoginForm() | ログイン画面を表示（view: auth.login） |
| login(Request $request) | email・passwordで認証。is_active=falseは拒否。成功時→/dashboard |
| logout() | セッション破棄→/loginへリダイレクト |

**バリデーション（login）:**
```
email: required, email, max:255
password: required, string, min:8
```

### 4.2 DashboardController

**ファイル:** `app/Http/Controllers/DashboardController.php`

| メソッド | 処理内容 |
|---------|---------|
| index() | 自分が発行したURL一覧（管理者は全件）を取得し表示 |

表示データ:
- 有効URL数、期限切れURL数、総ダウンロード数
- 直近10件のダウンロードURL（ファイル名・相手先・有効期限・DL数）

### 4.3 FileController

**ファイル:** `app/Http/Controllers/FileController.php`

| メソッド | 処理内容 |
|---------|---------|
| index() | アップロード済みファイル一覧を表示（自分のみ/管理者は全件） |
| store(Request $request) | ファイルをアップロード。storage/app/uploads/{user_id}/ に保存 |
| destroy(SharedFile $file) | ファイルを削除。deleted_files_log に記録。関連URLもsoftDelete |

**バリデーション（store）:**
```
files: required, array, min:1
files.*: file, max:102400（100MB）,
         mimes: jpg,jpeg,png,gif,pdf,doc,docx,xls,xlsx,ppt,pptx,zip,csv,txt
```

複数ファイルをループして SharedFile を1件ずつ作成する。

**ファイル保存パス:** `storage/app/uploads/{user_id}/{uuid}.{拡張子}`

**フロントエンド:** ファイル選択エリアはドラッグ＆ドロップ対応。アップロード中は進捗バーを表示（Fetch API + XMLHttpRequest.upload.onprogress を使用）。

### 4.4 DownloadUrlController

**ファイル:** `app/Http/Controllers/DownloadUrlController.php`

| メソッド | 処理内容 |
|---------|---------|
| index() | 発行済みURL一覧（自分のみ/管理者は全件）を表示 |
| create() | URL発行フォームを表示。アップロード済みファイル一覧もセット |
| store(Request $request) | URLを発行。tokenはStr::random(64)で生成 |
| show(DownloadUrl $url) | URL詳細・アクセスログを表示 |
| destroy(DownloadUrl $url) | URLをsoftDelete（ファイルは削除しない） |

**バリデーション（store）:**
```
shared_file_id: required, exists:shared_files,id
passcode: nullable, string, min:4, max:32
recipient_email: required, email, max:255
expires_at: required, date, after:now
download_limit: nullable, integer, min:1, max:9999
notify_on_download: boolean
```

### 4.5 DownloadController（相手先用）

**ファイル:** `app/Http/Controllers/DownloadController.php`

認証状態はセッションで管理する（`download_auth_{token}`キー）。

| メソッド | 処理内容 |
|---------|---------|
| showPasscode($token) | トークン検証→パスコード入力画面表示 |
| verifyPasscode($token, Request) | パスコード検証→OKならメールアドレス入力へ |
| verifyEmail($token, Request) | メールアドレス検証→OKならOTPを生成・送信 |
| showOtp($token) | OTP入力画面表示 |
| verifyOtp($token, Request) | OTP検証→OKならダウンロード画面へ |
| download($token) | ファイルをレスポンスとして返す。download_countを+1 |

**セッションキー（相手先認証状態）:**
```
download_auth_{token}.passcode_verified → true/false
download_auth_{token}.email_verified → true/false
download_auth_{token}.otp_verified → true/false
```

**OTP生成:**
- 6桁ランダム数字: `str_pad(random_int(0, 999999), 6, '0', STR_PAD_LEFT)`
- 有効期限: 発行時刻 + 10分
- 送信: Mail::to($recipient_email)->send(new OtpMail($code))

**ロック処理:**
- auth_codes.failed_count が 5回 → lock_until を現在時刻+30分にセット
- lock_until が未来の場合はOTP入力不可・エラー表示

**共通チェック（各メソッドの冒頭）:**
- トークンが存在しない → 404
- expires_at が過去 → エラービュー（有効期限切れ）
- download_limit に達している → エラービュー（上限超過）
- deleted_at がある → 404

### 4.8 Admin\SettingController

**ファイル:** `app/Http/Controllers/Admin/SettingController.php`

| メソッド | 処理内容 |
|---------|---------|
| index() | システム設定画面を表示 |
| update(Request) | 設定を保存（config/settings.json または settingsテーブルに保存） |

**設定項目:**
```

cleanup_grace_days: integer（有効期限後の猶予日数、デフォルト7）
notify_before_delete: boolean（削除前に担当者へ通知するか）
```

設定値は `storage/app/settings.json` に保存し、`config/site.php` 経由で参照する。

### 4.9 自動削除（Artisan Command）

**ファイル:** `app/Console/Commands/CleanupExpiredFiles.php`
**コマンド名:** `fileshare:cleanup`

**処理フロー:**
1. `expires_at` が猶予日数（デフォルト7日）以上過去の `download_urls` を取得
2. 削除前通知設定がONの場合、担当者に `DeleteNotificationMail` を送信（削除7日前）
3. 猶予期間を過ぎたファイルを `Storage::delete()` で削除
4. `deleted_files_log` に記録
5. 関連する `download_urls` を完全削除（softDeleteではなく物理削除）

**スケジューラー登録:** `app/Console/Kernel.php`
```php
$schedule->command('fileshare:cleanup')->dailyAt('02:00');
```

**Windowsでのスケジューラー実行:**
タスクスケジューラーに以下を登録：
```
C:\xampp\php\php.exe C:\xampp\projects\oneway-fileshare\artisan schedule:run
```
毎分実行に設定する。

## 7. エラー処理仕様

| 状況 | 処理 |
|------|------|
| トークンが存在しない | abort(404) |
| URLが期限切れ | download/error.blade.php に「有効期限切れ」を表示 |
| ダウンロード上限超過 | download/error.blade.php に「上限超過」を表示 |
| URLが無効化済み | abort(404) |
| パスコード不一致 | バリデーションエラーとして画面に表示 |
| メールアドレス不一致 | バリデーションエラーとして画面に表示（実在確認は不明示） |
| OTP不一致 | failed_countを+1。5回でロック。残り回数を表示 |
| OTPロック中 | lock_untilまでの残り時間を表示 |
| OTP期限切れ | 「コードの有効期限が切れました。再度URLにアクセスしてください」 |
| ファイルアップロード失敗 | バリデーションエラー表示 |
| ログイン失敗 | 「メールアドレスまたはパスワードが正しくありません」 |
| 非アクティブユーザーログイン | 「このアカウントは無効化されています」 |
